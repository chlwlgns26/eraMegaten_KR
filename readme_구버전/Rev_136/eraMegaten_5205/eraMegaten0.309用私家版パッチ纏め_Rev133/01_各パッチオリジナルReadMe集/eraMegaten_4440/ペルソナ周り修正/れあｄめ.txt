ペルソナ使いが多すぎてデータが壊れる原因はたぶんこれかな？と思い
ペルソナを管理している関数のDITEMTYPEの中身を100:100から300:100に変更

あと、COOP時のペルソナ経験値がもっと手に入りやすいように






たぶん、初期ペルソナのパラメータ入りはペルソナ使いが仲間に入ったときに関数に挿入してると思う

ペルソナはDITEMTYPEの第一引数でペルソナ種類が入り、第二引数にそのペルソナのパラメータが入ってる
共通ペルソナは10体までで0〜10に入っているので、初期ペルソナはたぶん10以上のどれかだと思う

たぶんエラーが出たのはペルソナ使い30人以上じゃなくて90人以上だと思う

第一引数を100から300にしたことによって1〜200まで全員ペルソナ使いでもデータが壊れることはない
と思います。
