

【スキル変化用関数_手直しパッチ】


【対象環境】
私家版まとめRev131


【導入方法】
１.そのままコピペ
２.『私家版追加関数』フォルダ内の『SKILL_CHANGE_CHECK.ERB』を削除


【このパッチについて】
このパッチはスキル変化用の関数『SKILL_CHANGE_CHECK』を手直しするパッチです。
……昔の自分が書いた拙いプログラムを見て我慢できなくなって直したパッチとも言う。


【変更点】
・処理の最適化
　何か色々無駄な処理とか変な処理をしていたので直しました。
　その結果、条件式の『 == 1』を省略できるようになりました。
　……何故-1を返すようにしていたのか。コレガワカラナイ

・関数名を『SKILL_CHANGE_CHECK』から『SKILL_CHANGE』に変更
　長ったらしいので。
　関数名を変えたので『SKILL_CHANGE_CHECK』を使っていたスキルを、関数の名前を変えた上で同梱します。
　当然ですが、処理等には一切手を加えていません。

　・この関数を作る前の、コメントアウトしていた処理を削除しました。
　エラーが出たらすぐに戻せるようにと残していましたがないようなので。


【関数の使い方】

IF SKILL_CHANGE(キャラの登録番号, キャラの固有番号)
    (処理)
ENDIF

登録番号は大抵、スキル用関数の引数の ARG に入っている筈
