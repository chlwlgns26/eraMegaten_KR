　パラメータなど

体力		左上の羽です。これが0になると死亡します。

満腹度		左上の青い棒です。技を使う際に消費します。

積載量		持てるアイテムの総重量です。

治癒力		体力の回復速度です。

近接スキル	近接攻撃ダメージに影響します。

射撃スキル	射撃攻撃ダメージに影響します。

その他武器スキル	その他の武器の攻撃ダメージに影響します。

作成スキル	投げナイフ、手榴弾、地雷の威力に影響します。レベルが上がるとオブジェクトなどを作成できるようになります。

料理スキル	料理使用時の回復量に影響します。

調合スキル	薬使用時の回復量に影響します。


　わかりにくそうなパラメーター

防御力		数値分の割合、ダメージを軽減します。

防御貫通		数値分防御力を無視します。

反動		レティクルの広がりに影響します。レティクルが広がると弾が正確に飛びません。

　システム


敵の中心あたりを攻撃するとクリティカルになり、ダメージが増えます。
敵の中心位置はカーソルを合わせると表示されます。


敵に気づかれていない状態で敵を攻撃すると、ダメージが増えます。


スニーク中は足音がしません。
草地や水たまりでも、大きな音を立てずに移動できます。


満腹度（青い棒）が0になると移動速度が低下します。


食べ物は腐敗します。腐る寸前に料理することで、実質的に鮮度を稼げますが、
別に気にせず適当に管理してもあまり変わらないと思うので、
このシステム自体忘れてもいいです。面倒だと思う人は忘れましょう。


腐敗した食べものは会得できる経験値が少なくなります。
また、料理の素材に指定できません。


敵は視覚、聴覚、嗅覚などを持っています。
視覚は敵から見て前方にいるプレイヤーを感知します。
草地に入っている間は敵から見えにくくなります。


聴覚はプーレヤーがたてた銃声や足音でプレイヤーを感知します。


嗅覚は敵の向きに関わらず、近づいたプレイヤーを感知します。
プレイヤーが水たまりに入ると、短時間嗅覚によって感知されない状態になります。


敵に気づかれた時に出る！マークは、周囲の敵を発覚状態にする効果があります。


技には再使用が可能になる時間と、攻撃不能になる時間が別々に設定されています。


敵を倒すと、攻撃力が短時間上がります。効果は時間とともに減少します。
効果量は画面の赤みで確認できます。


地図に表示されている赤い印は現在位置を示しています。
地図に表示されている青い印は行くべき場所を示しています。

　スクロールタイプの違い


　固定		カーソル位置によってスクロールしないので遊びやすく、正確なカーソル移動ができるが、縦の視界が狭い。あかんやん。

　半追尾		視界は広く、遊びやすいが、カーソル位置によって半端にスクロールするので、正確なカーソル移動ができない。おすすめ。

　手動		視界は広く、正確なカーソル移動ができるが、視界のコントロールが面倒。RTSプレイヤーなら気にならないかもしれない。

　完全追尾	酔う。非推奨。半追尾を正確なカーソル移動が可能なものに改良した気がするタイプ。酔わない人にはおすすめかもしれない。

　カスタムBGM

ゲーム内で使用される音楽を、プレイヤーが用意した音楽ファイルに差し替える機能です。
使用する場合は、よく読んでご使用ください。

Dataフォルダ内にある、CBGMフォルダに、ゲーム内で使われるBGMと差し替える音楽ファイルを入れ、
CBGMフォルダ内にある、CBGM.txtに、拡張子を含む音楽ファイルの名称を書くと、指定された音楽ファイルが使用されます。
BGM名と音楽ファイル名の間にはインデント（TABキーを押すと出る空白）を入れてください。
使用出来るファイル形式はmp3、ogg、midなどです。
もし指定したファイルが存在しない場合、デフォルトの音楽が鳴ります。


例

朝1	ほにゃらら.mp3

　以下どうでもいい話


このゲームでは小数点以下の数字は切り捨てられます。


ゲームの設定からBGMなどの音量を小さくし過ぎると音量関係の処理がバグります。
大体30以上にしておけば多分大丈夫です。


当たり判定はわりといい加減です。
特に極太レーザーの外見は信用しない方がいいです。


キーボードによっては3つ以上キーを押すと、キーの組み合わせによって反応しないので注意。
ゲーム用のキーボードなら問題ないらしいです。


ひとつのセーブデータで保存できるアイテムの最大数は39999です。
これ以上の数のアイテムが変数に格納されている場合、新規にアイテムが生成されるとバグるはずですが、
普通にプレイすれば39999もアイテムが保存されることはありえないと思います。
アイテムは消費されると情報が消滅するので、気になる人は消費したり売って消滅させてください。


ひとつのセーブデータで保存できるストレージ（小箱とか）の最大数は9999です。
これ以上の数のストレージが変数に格納されている場合、新規にストレージが生成されるとバグるはずですが、
普通にプレイすれば9999もストレージが保存されることは絶対ありえないと思います。やろうと思っても無理だと思う。
ストレージは分解されると情報が消滅するので、気になる人は分解して消滅させてください。


System.savは消さないでください。


セーブデータを消去する際はsaveフォルダ内から手動で消してください。
セーブデータロード画面の、左上がSaveData1.savで、
下に行くほど数字が大きくなります。
例えば、セーブデータロード画面の、左上からひとつ下のデータは、SaveData2.savです。


作者はシューティングゲームが苦手かつ嫌いなので、
このゲームがまともにシューティングゲームとして遊べるようにできているのかどうか知らない。
わりと被弾前提。ノーダメクリアはほぼ不可能だと思う。


一部のスキルが使用されている間にセーブすると、多分ロード時にエラーを吐く。
技を使用中にはセーブしない方がいいが、普通しないと思うので気にしてない。


一部のボスは音が聞こえないと戦いにくいです。
聴覚障害の方には少し辛いかもしれません。


使用しているウディタは2.10です。
このゲームは2.20系列だと効果音が上手く読み込めないバグがでます。


ゲームを終了する		Esc っていうコマンドがあったのですが、
不評なようなので消滅させました。Alt+F4で終了してください。
パソコン初心者の人が終了できるのか不安ではある・・・。


たまにアップデートでバグが増えるかもしれません。増えたらごめんね。
