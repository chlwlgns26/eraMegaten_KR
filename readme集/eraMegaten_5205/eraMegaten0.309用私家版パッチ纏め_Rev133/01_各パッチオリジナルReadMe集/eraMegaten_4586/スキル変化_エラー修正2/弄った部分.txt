・SKILL_CHANGE_CHECK.ERB
　新規作成。
　@SKILL_NAME 等の引数(ARG)がキャラ登録番号の他に装備品の番号とかペルソナのキャラ登録番号だったりする場合がある。それを判別する為に条件式を書いたら長くなったので関数にした。
　関数内に色々書いているのは『ペルソナや装備品でもスキルを変化させてぇなぁ』という願望の名残。
　第１引数は調べる数値。第２引数は、第１引数と同じかどうか判定したい数値。あとこの関数は式内で使うもので、CALL などの通常の呼び出しは不可。

　Ｑ．つまり？
　Ａ．『IF SKILL_CHANGE_CHECK(ARG, 特定のキャラの番号) == 1』って書いておけばおｋ。


・多数のスキル.ERB
　条件式を上記の関数に変更。
　NO:XXX だとARGに所持しているキャラの数より大きい数値を入れられるとエラーになります。上記関数の説明で書いた通り、装備品の番号(2000〜)やペルソナのキャラ登録番号(50〜)が入る場合があるので変更させて頂きました。

　ゲシュペンスト等ＰＴ、レヴュースタァライト、ナナドラ、首斬り破沙羅のスキルは調査済みです。
