○修正内容
resources
　サブフォルダを作成して一部の画像とcsvを移動
　resources
　┠画像.csv
　┠画像.csvに記述されている画像ファイル
　┠画像_EX
　┃┠画像_EX.csv
　┃┗画像_EX.csvに記述されている画像ファイル
　┠幻想郷
　┃┠幻想郷.csv
　┃┗幻想郷.csvに記述されている画像ファイル
　┗アイコン
　　┠アイコン.csv
　　┗アイコン.csvに記述されている画像ファイル

　以上のように画像.csvのcsvとそれに記述されているファイルは
　csv名のサブフォルダを作成してそこにいれています

WRITE_IMG.ERB
　画像関連処理用の関数を作成
　WRITE_IMG
　→ WRITE_IMG_1,WRITE_IMG_2の機能を統合
　　 呼び出すリソースが存在するか判定して
　　 リソースが存在しなければ図形を出力する
　　 （ここに到達する前に簡易リソース判定により
　　　アイコン出力などに切り替えられているはずなので
　　　この時点でリソースが存在しないのはイレギュラー）

　GET_IMG_TYPE
　→　伽耶ちゃん判定機
　　　WRITE_IMG_1,WRITE_IMG_2,SHOW_IMG_S,SHOW_IMG_Lから
　　　例外処理を行うキャラクターの判定部分を移行
　　　例外処理を画像種別としてリソースと紐付けるようにして汎用化した

　SHOW_IMG
　→　SHOW_IMG_S,SHOW_IMG_Lの機能を統合
　　　調教対象と助手の画像出力用
　　　画像がない場合の大体処理の判定もここ


FORMATION.ERB
　伽耶ちゃん判定を外だし、csvの内容変更に対応

SHOP.ERB
　SHOW_IMG_S,WRITE_IMG_1を削除
　SHOW_IMG,WRITE_IMGを呼び出すように変更

INFO.ERB
　SHOW_IMG_L,WRITE_IMG_2を削除
　SHOW_IMG,WRITE_IMGを呼び出すように変更

SHOW_STATUS.ERB
  WRITE_IMGを呼び出すように変更

EXIST_PICTURE.ERB
　画像対応しているキャラクター番号を直接記述するのではなく
　キャラクターのリソースが登録されているかどうかの判定に変更

EVENT_52_EXTRA.erb
EVENT_52_変わった悪魔の売買.ERB
ICON.ERB
　csvの内容変更に対応
